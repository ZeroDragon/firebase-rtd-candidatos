Prueba Resuelve

Bienvenido a las pruebas para candidato Resuelve.
Estas pruebas fueron diseñadas para evaluar la habilidad del desarrollador de frontend o backend.
Cada sección tiene su propio reto pero si sabes como abstraer el problema es realmente fácil resolver los ejercicios.
Para el ejercicio de frontend, no esperamos que seas un disegramador, pero si esperamos que tenga cierta estética (usar un framework de css es válido).
Para el ejercico de backend, puedes usar cualquier lenguaje de programación el cual te sea más cómodo, sin embargo, debes incluir instrucciones de cómo montarlo, echarlo a andar y probarlo funcionando.

Criterios de evaluación

Cada prueba trae sus instrucciones propias, aparte de eso esperamos que cuente con lo siguiente:

  1. Tu código debe estar en un repositorio git (github, gitlab, bitbucket)
  2. Debe contener un README.md donde explique cómo montar tu servicio y cómo correrlo para revisarlo

Si tu código hace lo mínimo requerido está bien, pero todo lo extra que quieras agregar se evaluará y tomaremos en cuenta cosas como:

  1. Limpieza de código (que no sea un batidero)
  2. Comentarios (que se entienda que hace cada sección)
  3. Pruebas unitarias
  4. Emojis, unicornios, arcoiris, etc

Sin miedo a equivocarse

Nosotros en Ingeniería de Resuelve no tenemos miedo a equivocarnos, lo vemos como un método de aprendizaje, recuerda que mientras aprendas, no es tiempo perdido.
Así que no te agobies si no logras terminar toda la prueba (obvio es lo ideal xD). Mándanos tu mejor trabajo, nosotros tomaremos en cuenta todo lo que hiciste, tu forma de escribir código, tu forma de comentar, si parametrizaste la mayoria. Todo es parte de ti y nos interesa conocerlo, nadie nace sabiendo y si eres alguien adecuado nos gustaría que aprendieras con nosotros.

Ejercicio Frontend

¿Cuál es el problema?

  1. Nuestros clientes necesitan tener un lugar donde puedan hacer login con su usuario y contraseña para poder ver sus cuentas
  2. Necesitamos un acceso administrador donde se puedan ver todos los usuarios y sus cuentas

¿Qué es lo que debo hacer?

  1. Crear un aplicación web en frontend usando la librería que más te guste (jquery, angular, react, vue, backbone, etc)
  2. En la app debe haber un login para usuarios y un login para administradores
  3. Debes manejar JWT para las secciones protegidas (Documentación)
  4. Algunos usuarios están desactivados y regresarán un error al hacer login
  5. Del lado del admin debe poder ver un listado de clientes y por cada cliente un listado de movimientos
  6. Del lado del cliente debe poder ver un listado de sus movimientos
  7. Todos los listados deben tener un paginador y poder ordenar los elementos por cualquier columna por su nombre
  8. En el caso del listado de movimientos, debe haber un botón para convertir los pesos en dólares y viceversa

Material de apoyo

Sugerimos primero hagas la sección del admin para que puedas observar los usuarios existentes, cuáles son sus correos y cuáles están activos o desactivados.
Posteriormente hacer la sección de clientes, la cual con lo anterior resuelto es solo cambiar los endpoints y reutilizar componentes.

Para poder lograr esto, se hizo un API donde puedes consultar todo lo necesario:

API URL https://us-central1-prueba-resuelve.cloudfunctions.net

Endpoint /users/login

Para hacer login con cualquier usuario

Payload
axios.post('API_URL/users/login', {
  email: 'email del usuario',
  password: 'password del usuario'
})

})
.catch(function (error) {

Ejercicio Backend

¿Cuál es el problema?

  1. Hay que reportar todos los gastos de 2018 al área de finanzas, ellos esperan una estructura específica con un resumen de los gastos y un desglose por cada cliente.
  2. El API de nuestro sistema tiene un lugar donde regresar los datos de clientes y movimientos, pero solo por rangos de fechas y si trae más de cincuenta elementos contesta con un error.
  3. El sistema que nos da la información está muy limitado en costos, por lo tanto es muy importante no hacer llamadas innecesarias al servicio (esto solo durante la ejecución de tu programa, mientras estás desarrollando no te limites)

¿Qué es lo que debo hacer?

  1. Obtener un listado de todos los clientes (los cuales se dieron de alta en 2017)
  2. Obtener un listado de todos los movimientos generales (creados durante el 2018)
  3. Generar un resumen de todos los datos generales
  4. Generar un resumen por cada cliente con la suma de sus movimientos
  5. Armar un solo resultado que incluya ambos resúmenes con la estructura de datos esperada

Material de apoyo

El objetivo de este servicio es que sea lo más rápido posible y que use la mínima cantidad de llamadas al API de datos
Genera un log de cuantas llamadas hiciste para obtener los datos y cuánto tiempo toma obtener toda la información.

}
Notas:
  • Este endpoint es muy estricto en la forma en que se deben recibir los valores, el formato esperado es el siguiente:

  "totalRecords": total_de_movimientos,
  "totalCredit": suma_del_credit,
  "totalDebit": suma_del_debit,
  "balance": credit - debit,
  "byUser": [
    {
      "name": nombre_del_usuario,
      "uid": uid_del_usuario,
      "records": total_de_movimientos_del_usuario,
      "resumen": {
        "credit": suma_del_credit_de_este_usuario,
        "debit": suma_del_debit_de_este_usuario,
        "balance": credit - debit
      }
    },
    {
      "name": otro_usuario,
      "uid": uid_del_usuario,
      "records": total_de_movimientos_del_usuario,
      "resumen": {
        "credit": suma_del_credit_de_este_usuario,
        "debit": suma_del_debit_de_este_usuario,
        "balance": credit - debit
      }
    },
    ...
  ]
}
